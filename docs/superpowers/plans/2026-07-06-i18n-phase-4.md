# i18n Phase 4 — Finance Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development, task-by-task.

**Goal:** Fully bilingual Finance — invoices, quotes, contracts, credit-notes, expenses, purchase-orders, suppliers, recurring, targets, client-statement, finance overview + reports tabs, and all 38 finance API routes — Arabic pixel-identical; money formatting byte-identical in both locales.

**Ground truth inventories:** `.superpowers/sdd/p4-discovery/p4-{invoices-quotes,invoice-subcomponents,finance-pages-a,finance-pages-b,reports,finance-apis,crosscut}.txt` — every AR string verbatim with line numbers, ICU flags, class tags (ui/data/comment/statuses; a/b/c/pdf/email for APIs), label-drift tables, and the accessor swap-target list.

## Global Constraints

All Phase 0–3 constraints hold (AR verbatim; D7 fallback; NO JSON arrays in catalogs; digit-glyph preservation via in-place `'ar-EG'↔'en-GB'` parametrize; `localeCompare(x, locale)`; ICU `other` reproduces today's AR; flat-threshold keys instead of ICU plural when reproducing an AR threshold ladder — P3-T6 lock; bilingual DB rows render `locale==='ar' ? name_ar : (name || name_ar)`; per-task `pnpm run check` + `pnpm test` + `pnpm build`; commit trailer `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`; no push until closure). Phase-4-specific:

- **Money formatting is LOCALE-INVARIANT by design.** `formatCurrency` has no locale param and renders ASCII digits via `en-AE` (crosscut §2, verified empirically). Do NOT add a locale param, do NOT localize digits in money, do NOT touch hardcoded `'AED'` currency-code displays. Exit criterion: money renders byte-identical in AR and EN.
- **DB-persisted content stays Arabic + `// i18n-exempt: stored data`:** credit-note auto-fill `reason` template (`credit-notes/new/page.tsx` L54), recurring-invoice auto-titles incl. `cycleLabels` map (`app/api/finance/contracts/[id]/route.ts`), invoice/recurring notes, activity descriptions, the Arabic-alphabet ordinals in `lib/finance/recurring-generation.ts` L178. The `contract-items-editor.tsx` L124 Arabic ordinals (أ ب ج…) also stay — they flow into invoice items consumed by clients (recipient-language = Phase 9).
- **Outbound artifacts are Phase 8/9 EXEMPT:** notification content (4 lines), email subject/body (2 lines), CSV export route (`app/api/reports/export/route.ts` — all-Arabic headers + نعم/لا cells stay; backlog entry for Phase 9 locale threading). The `.toLocaleDateString('en-CA')` ×6 in the export route are ISO-machine-format — MUST NOT be localized.
- **Deferred out of Phase 4:** `app/dashboard/reports/**` + `app/api/reports/**` (separate module, `reports.view` gate — Phase 6), `app/dashboard/finance/cards/page.tsx` (orphaned subscriptions-sunset page — do not migrate; manifest lists finance subdirs explicitly so cards stays outside; removal decision is a pending user chip), cron/webhook routes.
- **Label-drift reconciliation: canonical = `lib/constants/statuses.ts` → statuses.json** (they match; extracted verbatim in Phase 1). Local divergent maps converge to canonical on swap — each is a documented pre-existing drift bug fix, called out per task: quote `signed` «موقع→موقّع» (quotes-analytics), creditNote `applied` «مطبق/مطبّق», expense `pending` double-phrasing, paymentMethod `cash` variants, PO page's copy-pasted CONTRACT vocabulary → real `po` labels (functional bug fix — commit message must say so).
- **New/extended statuses entities (T1):** `contract` gains `in_progress` = «قيد التنفيذ» (finance's live wording; `crmContract`'s «جاري» is a separate entity and untouched — verbatim-per-surface rule); new `contractType` entity (retainer/milestone/upfront_delivery/fixed/hourly — AR verbatim from finance pages); new `periodCycle` entity (monthly/quarterly/yearly — the triad duplicated 6× across 4 files; AR = the canonical BILLING_CYCLE wording). `expense.draft` gap: UI-only value — give it a key with the UI's current wording.
- **Bilingual-field pattern:** `QuoteBuilder.tsx` L576 `{t.name_ar || t.name}` → `locale === 'ar' ? (t.name_ar || t.name) : (t.name || t.name_ar)`.
- **Sort comparators:** `localeCompare(x, 'ar')` ×4 (invoices-client L212-213, quotes-client L199-200, contracts-client, expenses-client) → `localeCompare(x, locale)`.
- **Structural dir fix:** `app/dashboard/finance/recurring/new/page.tsx` root `dir="rtl"` → `dirFor(locale)`. No drag machinery exists anywhere in finance — no lock map this phase.
- **Server-computed month/chart labels:** T1 adds `MONTH_NAMES_EN` + `monthNamesFor(locale)` next to the canonical `MONTH_NAMES_AR`; pnl/cashflow/vat routes + `app/api/finance/dashboard/route.ts` L147 (`toLocaleDateString('ar-EG')`) resolve the caller's locale via `getLocale()` from `next-intl/server` (cookie-driven, same plumbing as `getTranslations`) and build labels per request. AR output byte-identical. The pnl route's inline quarter-name array gets the same treatment (AR verbatim).
- **Recharts threading:** the 3 chart components + finance-reports tabs hardcode Arabic inside Tooltip/Legend/formatter closures — `useTranslations` must be called at component top and the resolved strings threaded into the closures (closures can't call hooks).
- **`MAP[key] || rawKey` fallback-leak pattern (×5 in invoice sub-components):** preserve the raw-key fallback through migration (`statusLabelFor(key) || key`) — never let an unknown DB enum render blank.
- **Hands-off:** raw-fetch/useState data-loading in finance pages is OUT OF SCOPE (documented backlog) — implementers must NOT convert data fetching to React Query in this phase. window.prompt() in QuoteBuilder stays (structural).
- **Manifest (closure task only):** append per-subdir paths — `app/dashboard/invoices`, `app/dashboard/quotes`, `app/dashboard/finance/client-statement`, `.../contracts`, `.../credit-notes`, `.../expenses`, `.../purchase-orders`, `.../recurring`, `.../reports`, `.../suppliers`, `.../targets`, `app/dashboard/finance/page.tsx`, `components/quotes`, `components/finance`, `components/dashboard/invoice-detail`, `components/dashboard/invoice-new`, `components/dashboard/client-statement`, `components/dashboard/finance-overview`, `components/dashboard/finance-reports`, `components/reports`, `app/api/invoices`, `app/api/quotes`, `app/api/finance`, the 7 finance hooks. NOT `app/dashboard/finance` bare (cards would fail), NOT `app/dashboard/page.tsx`… unless T8 lands it clean — then add it.
