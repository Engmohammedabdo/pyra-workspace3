# i18n Phase 2 — Work Core (My Work, My Tasks, Boards, Calendar) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Make the daily-work surfaces fully bilingual — My Work inbox, my-tasks, the entire boards/tasks stack (incl. task-sheet), the calendar stack, and their API response messages — with Arabic pixel-identical and the locked drag machinery untouched.

**Architecture:** Same recipe as Phases 0–1: per-namespace catalogs (`mywork`, `boards`, `calendar`, `api`), `useTranslations`/`getTranslations`, `useStatusLabels` for label maps, locale params threaded into format helpers. New this phase: the shared `api` namespace for server response messages (dedup'd), a `taskPriority` + `leaveType` statuses entity, a structural `kind` field on `formatTaskDueDate`, and deletion of the dead `TaskDetailDialog` (~1030 lines).

**Tech stack:** unchanged (next-intl 4.x, vitest, pnpm).

**Ground truth:** the six discovery inventories at `.superpowers/sdd/p2-discovery/p2-{mywork,boards-view,boards-components,calendar,apis,cross-cutting}.txt` list EVERY Arabic string verbatim with line numbers, every ICU-worthy string, every `dir` attr, every hardcoded locale call. Implementer briefs cite them — they are the requirements detail; the SOURCE FILES are the final truth for AR values.

## Global Constraints

All Phase 0–1 constraints remain (AR verbatim extraction, D7 fallback, `pnpm run check`+`test`+`build` green per task — note `check` now RUNS i18n:check, one namespace per file, commit trailer, no push until closure task). Phase-2-specific:

- **AR pixel-identical, hard mode:** the boards are in daily production use (video team). Preserving AR output includes DIGIT GLYPHS: replace hardcoded `toLocaleDateString('ar-EG', opts)` with `toLocaleDateString(locale === 'ar' ? 'ar-EG' : 'en-GB', opts)` (same opts object, in place) — do NOT reroute those calls through date-fns helpers (that would silently change Arabic-Indic digits to Western). Same for `localeCompare(x, 'ar')` → `localeCompare(x, locale)`.
- **DRAG MACHINERY IS LOCKED.** In `board-view-client.tsx`: do NOT touch `closestCorners` (L1998-region), the `dir="ltr"` scroll containers (L2003 kanban, L2260 pipeline), sensors, any `useSortable`/`useDroppable` wiring, or `handleDragEnd` logic. The ONLY i18n-touchable item inside drag code is the toast STRING at ~L1800. The three inner `dir="rtl"` compensation spans (L2276, L2299, L2329) become locale-conditional: `dir={locale === 'ar' ? 'rtl' : undefined}`.
- **Dead code is deleted, not translated:** `TaskDetailDialog` in board-view-client (comment banner ~L523 through function end ~L1553) is confirmed never-rendered — Task 6 removes it entirely. Never seed catalogs from its strings.
- **API string classes** (per `p2-apis.txt` census): class (a) API response messages → translate NOW via `getTranslations('api')`; class (b) notification/WhatsApp content → STAYS Arabic + `// i18n-exempt: notification content (Phase 8)`; class (c) DB data writes (comment bodies, attachment names, seeded column names) → stays Arabic + `// i18n-exempt: DB data`; class (d) the `nextCol.name === 'معتمد'` business comparison in approve/route.ts → DO NOT TOUCH (pre-existing hazard, documented below).
- **URL/state contract keys stay English:** calendar `?view=`/`?types=` values, priority keys, groupBy keys — only display labels localize.
- **Response-helper DEFAULTS stay:** `apiUnauthorized()`/`apiServerError()` no-arg defaults in `lib/api/response.ts` are a cross-phase concern (sync helpers; localizing them is a dedicated later pass) — out of scope.
- **`components/calendar/calendar-event-pill.tsx` local `formatTime` is LOCKED** (pre-zoned +04:00 slicing, HR v1.1 cleanup decision) — output is ASCII HH:MM, locale-neutral; add `// i18n-exempt: locale-neutral time slice (locked local copy)` if the scanner ever flags its comments, otherwise leave.
- **Coordination:** a separate session (chip task_b73f2afc) may land changes to `app/api/tasks/[id]/move/route.ts` + `lib/evolution/client.ts`. Task 8 (API routes) runs LAST and starts with `git fetch && git pull --ff-only` + re-reading that file.
