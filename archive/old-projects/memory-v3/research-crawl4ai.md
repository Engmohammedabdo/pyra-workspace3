## PDFKit Arabic Support

### هل يدعم RTL؟

**❌ لا يوجد دعم RTL مباشر في PDFKit.**

من تحليل الوثائق الكاملة لـ `pdfkit.org/docs/text.html`:

- **الخيارات المتاحة للنص:**
  - `align`: left, center, right, justify — لكن هذا alignment وليس text direction
  - `features`: OpenType feature tags — يمكن تفعيل `rtla` (Right-to-Left Alternates) لكن ليس RTL layout كامل
  - لا يوجد خيار `direction: 'rtl'` أو `bidi`
  
- **الخطوط:** يدعم TrueType (.ttf), OpenType (.otf), WOFF, WOFF2 — يعني ممكن تستخدم خط عربي
  
- **المشكلة الأساسية:**
  - PDFKit لا يعالج BiDi (Bidirectional text)
  - الحروف العربية ما تتصل ببعض (تطلع منفصلة)
  - الترتيب من يسار لـ يمين بدل يمين لـ يسار

### البدائل لـ Arabic PDF:

| البديل | RTL Support | ملاحظات |
|--------|-------------|---------|
| **puppeteer + HTML** | ✅ كامل | الأفضل — اطبع HTML عربي كـ PDF عبر Chrome |
| **jsPDF + arabic-reshaper** | ⚠️ جزئي | يحتاج reshaper خارجي |
| **React-PDF (@react-pdf/renderer)** | ❌ | نفس مشكلة PDFKit |
| **wkhtmltopdf** | ✅ كامل | CLI tool يحول HTML لـ PDF |
| **Playwright PDF** | ✅ كامل | مثل Puppeteer |

### التوصية:
**استخدم Puppeteer/Playwright لتحويل HTML عربي لـ PDF.** أسهل وأضمن من PDFKit.

```javascript
// مثال Puppeteer Arabic PDF
const browser = await puppeteer.launch();
const page = await browser.newPage();
await page.setContent(`
  <html dir="rtl" lang="ar">
  <head><style>body { font-family: 'Noto Sans Arabic'; }</style></head>
  <body><h1>مرحباً بالعالم</h1></body>
  </html>
`);
await page.pdf({ path: 'output.pdf', format: 'A4' });
```

## OpenClaw Docs — Context Management

### Compaction (ضغط السياق)

**نعم، فيه نظام compaction متقدم!**

#### ما هو Compaction؟
- يلخص المحادثات القديمة في ملخص مضغوط
- يحتفظ بالرسائل الأخيرة كاملة
- يحفظ الملخص في JSONL history

#### الإعدادات:
```json5
{
  agents: {
    defaults: {
      compaction: {
        reserveTokensFloor: 20000,
        memoryFlush: {
          enabled: true,
          softThresholdTokens: 4000,
          systemPrompt: "Session nearing compaction. Store durable memories now.",
          prompt: "Write any lasting notes to memory/YYYY-MM-DD.md; reply with NO_REPLY if nothing to store."
        }
      }
    }
  }
}
```

#### أوامر مفيدة:
- `/compact` — ضغط يدوي (مع تعليمات اختيارية)
- `/compact Focus on decisions and open questions`
- `/status` — عرض عدد الضغطات
- `/context list` — تفصيل ما يتم حقنه
- `/context detail` — تفصيل أعمق

### Session Pruning (تقليم الجلسة)

- **مختلف عن Compaction** — يقلم tool results القديمة فقط
- **في الذاكرة فقط** — لا يغير الـ JSONL على القرص
- **يعمل مع Anthropic فقط** (cache-ttl mode)

#### إعدادات Pruning:
```json5
{
  agent: {
    contextPruning: {
      mode: "cache-ttl",  // أو "off"
      ttl: "5m",
      keepLastAssistants: 3,
      softTrimRatio: 0.3,
      hardClearRatio: 0.5
    }
  }
}
```

#### Soft vs Hard Pruning:
- **Soft-trim**: يبقي الأول والأخير من tool result، يحط `...` بالنص
- **Hard-clear**: يستبدل كل tool result بـ `[Old tool result content cleared]`

### Memory (الذاكرة)

- **ملفات Markdown عادية** — `memory/YYYY-MM-DD.md` + `MEMORY.md`
- **أدوات بحث**: `memory_search` (semantic) + `memory_get` (targeted read)
- **Vector search** مدعوم — embeddings عبر OpenAI/Gemini/Voyage/Local
- **QMD backend** (تجريبي) — BM25 + vectors + reranking محلي

#### Pre-compaction Memory Flush:
- **ميزة ذكية!** — قبل الضغط، OpenClaw يعطي الـ agent فرصة يحفظ ملاحظات مهمة
- يشتغل تلقائياً لما الـ session تقترب من حد الـ context window

### Context — ما يتم إرساله للـ Model:

| المكون | الوصف |
|--------|-------|
| System Prompt | قواعد + أدوات + skills + وقت + runtime |
| Project Context | AGENTS.md, SOUL.md, TOOLS.md, USER.md (max 20K chars/file, 150K total) |
| Conversation History | رسائل المستخدم + الـ assistant |
| Tool Calls/Results | نتائج الأدوات |
| Compaction Summaries | ملخصات الضغط |

#### أوامر مراقبة:
- `/status` — نظرة سريعة على امتلاء النافذة
- `/context list` — تفصيل الملفات المحقونة
- `/context detail` — أحجام كل tool schema + skill
- `/usage tokens` — footer بالاستخدام لكل رد
